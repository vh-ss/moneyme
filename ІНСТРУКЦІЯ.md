# MoneyMe — повна інструкція

Персональний застосунок обліку фінансів: рахунки, витрати й надходження за статтями, перекази з
комісією та курсом, бюджети, кредити, цілі, крипто, регулярні платежі, аналітика. Працює **офлайн**,
з опційною **синхронізацією через Google Drive** (ПК ↔ телефон).

---

## 1. Що це за проєкт (важливо зрозуміти структуру)

У папці насправді **дві реалізації** того самого застосунку:

| Що | Файл/тека | Призначення |
|----|-----------|-------------|
| **Канонічний застосунок** (актуальний) | `MoneyMe.html` | Односторінковий офлайн-застосунок. Усе в одному файлі, дані — у `localStorage` браузера. Це те, що реально використовується й розгортається. |
| Демо-пісочниця | `MoneyMe-DEMO.html` | Той самий застосунок із вшитими демо-даними (`demo-2026.json`). Ізольований простір `moneyme.demo.*`, ваші реальні дані не чіпає. |
| Точка входу для хостингу | `index.html`, `sw.js`, `manifest.webmanifest`, `icons/` | Копія застосунку + PWA-обгортка для GitHub Pages (іконка, офлайн-кеш, версіонування). |
| Вихідне монорепо (рання версія) | `server/` + `web/` | Express + SQLite + React/Vite. Окрема, менша за можливостями реалізація. Потребує Node.js. |

> **Головне:** робочий застосунок — це `MoneyMe.html`. Монорепо `server/web` — це історична/паралельна
> гілка з вужчим набором функцій (без кредитів, крипто, цілей, бюджетів, синхронізації).

Середовище користувача — **без Node.js та без інтернету**, тому код правлять напряму у `MoneyMe.html`,
а перевіряють, відкриваючи цей файл у браузері.

## 2. Швидкий старт (для користувача)

### Варіант А — просто відкрити файл (офлайн, локально)
1. Двічі клацніть `MoneyMe.html` — відкриється у браузері (Chrome/Edge).
2. Усе працює одразу: дані зберігаються в цьому браузері (`localStorage`, ключ `moneyme.v1`).
3. Для безпеки робіть **Експорт JSON** у Налаштуваннях (резервна копія).

> Обмеження локального файлу: **немає синхронізації через Google Drive** (Google дозволяє вхід лише
> зі сторінок `https://…`, не з `file://`). Для синхронізації потрібен хостинг — див. розділ 6.

### Варіант Б — спробувати на демо-даних
Відкрийте `MoneyMe-DEMO.html` — побачите застосунок, заповнений прикладами операцій за 2026 рік.
Ваші справжні дані недоторкані (демо живе в окремому просторі).

## 3. Розділи застосунку

Навігація (зліва/зверху): **Огляд · Операції · Аналітика · Рахунки · Кредити · Крипто · Цілі ·
Регулярні · Категорії · Налаштування**.

### Огляд (Dashboard)
Загальний баланс по всіх рахунках, швидке додавання витрати/надходження, прогрес бюджетів,
найближчі регулярні платежі та платежі за кредитами, прогноз балансу.

### Операції (Transactions)
Список усіх записів із фільтрами (рахунок, категорія, тип, період). Типи записів:
- **Витрата** (`−`) — списання з рахунку за статтею.
- **Надходження** (`+`) — зарахування на рахунок за статтею.
- **Переказ** — між двома рахунками з **комісією** (% або фіксована) і **курсом** (для різних валют).
  Комісія ведеться окремо й **не змішується з витратами**.
- **Split-операції** — наприклад, спільна покупка: записуються повна витрата + повернення, у
  бюджет/категорію йде лише ваша частка.

Переказ можна зробити **перетягуванням** картки одного рахунку на інший.

### Рахунки (Accounts)
Готівка / Картка / Банк, у валютах **UAH, USD, EUR, PLN, GBP**. У кожного — стартовий залишок,
колір, іконка. Поточний баланс рахується автоматично:
`стартовий + надходження − витрати − (перекази-out + комісії) + перекази-in`.

### Категорії (Categories)
Статті витрат і надходжень. У витратних можна задати **бюджет** (місячний або річний) з опцією
**переносити невитрачене** на наступний період. На Огляді/Аналітиці показано прогрес і перевитрати.

### Кредити (Credits / loans)
Облік боргів: залишок, щомісячний платіж, прив'язка операцій-платежів до кредиту.

### Крипто (Crypto)
Портфель монет. Ціни автопідтягуються з **CoinGecko** (за наявності інтернету); для невідомих монет —
ручна ціна. Можна задати **цінові сповіщення** (вище/нижче порогу).

### Цілі (Goals)
Накопичувальні цілі з прив'язаним рахунком і необов'язковим дедлайном.

### Регулярні (Recurring)
Повторювані платежі/підписки: сума, період, дата наступного списання. Застосунок нагадує, що
час внести, і показує прострочені.

### Аналітика (Analytics)
Суми за статтями, зведення за період (витрати / надходження / **комісії окремо** / чистий потік),
помісячна динаміка, витрати за тегами, прогноз балансу.

### Налаштування (Settings)
Мова (укр/eng), тема оформлення, валюта/курси, **резервне копіювання** (Експорт/Імпорт JSON),
**точки відновлення**, синхронізація з Google Drive, очищення даних.

## 4. Дані, бекап і безпека

- **Сховище:** усе локально в `localStorage` браузера (ключ `moneyme.v1`). Дані не виходять за межі
  пристрою, доки ви не ввімкнете синхронізацію.
- **Курси валют:** НБУ (`bank.gov.ua`) — за наявності інтернету; інакше беруться останні збережені.
- **Експорт/Імпорт JSON:** повний дамп даних у файл і відновлення з нього. **Імпорт замінює** поточні дані.
- **Точки відновлення:** раз на день застосунок зберігає знімок стану (до 7 останніх) — можна
  відкотитися, якщо щось пішло не так.

> Регулярно робіть Експорт JSON — це ваша надійна резервна копія поза браузером.

## 5. Синхронізація ПК ↔ телефон (Google Drive)

Працює лише з хостингу (`https://…`). Коротко (повний покроковий гайд — у [SETUP.md](SETUP.md)):

1. **Розмістити** вміст папки на **GitHub Pages** (файли `index.html`, `sw.js`,
   `manifest.webmanifest`, тека `icons/`, а також `MoneyMe.html`). Отримаєте адресу-**БАЗУ**.
2. **Створити Google OAuth Client ID** (Google Cloud Console → Drive API → OAuth consent screen
   *External/Testing* → ваш акаунт у тест-юзери → Credentials → Web application → у *Authorized
   JavaScript origins* додати ваш origin без шляху).
3. У застосунку (**Налаштування → Синхронізація**) вставити **Client ID** → **Підключити Google Drive**
   → увійти тим самим акаунтом. Те саме повторити на телефоні (бажано «Додати на головний екран»).

Особливості:
- **Офлайн-first:** дані миттєво локально; Drive — спільне хмарне сховище, доганяє при появі мережі.
- **Конфлікти:** перемагає новіша версія за міткою часу; версію, що програла, збережено в Точках відновлення.
- **Приватність:** scope `drive.appdata` — застосунок бачить **лише свій** файл (`moneyme-data.json`
  у прихованій папці застосунку), не решту вашого Drive. **Client ID не є секретом.**

## 6. Розробка та збірка

### 6.1 Канонічний застосунок (`MoneyMe.html`)
- Правиться **напряму** у `MoneyMe.html` (один файл: HTML + CSS + JS).
- Перевірка — відкрити файл у браузері. `APP_VERSION = 'dev'` у вихіднику; реальну версію
  підставляє збірний скрипт.
- i18n: рядки укр/eng у таблицях перекладів усередині файлу. Перевірити покриття перекладів —
  скрипт `.scan-i18n.cjs`.

### 6.2 Збірні скрипти (потрібен Node.js)
| Скрипт | Що робить |
|--------|-----------|
| `node .build-pwa.cjs` | Запускає юніт-тести (`tests.cjs`); якщо ок — генерує `index.html` (копія застосунку з версією = хеш збірки), штампує версію у `sw.js`, перезбирає демо. **Падає, якщо тести зламані.** |
| `node .build-demo.cjs` | Збирає `MoneyMe-DEMO.html`: префіксує ключі `moneyme.* → moneyme.demo.*` і вшиває демо-дані з `demo-2026.json`. |
| `node .gen-demo.cjs` | Генерує демо-дані. |
| `node .gen-icons.cjs` | Генерує іконки PWA. |
| `node tests.cjs` | Юніт-тести доменної логіки (комісія, курс, баланси). |
| `node .scan-i18n.cjs` | Перевіряє повноту перекладів. |

> Версія збірки = SHA-256 від `MoneyMe.html` + іконок + manifest. Будь-яка зміна ассета бастить кеш
> service worker, тож браузер бачить оновлення.

### 6.3 Вихідне монорепо (`server/` + `web/`, рання версія)
Потрібен Node.js 18+ та інтернет для першого `npm install`:
```bash
cd MoneyMe
npm install        # ставить залежності server і web
npm run dev        # API → http://localhost:4000, фронт → http://localhost:5173
```
Інші скрипти: `npm run dev:server` / `npm run dev:web`, `npm run typecheck`, `npm test`.
Дані — у `server/data/moneyme.db` (SQLite). Бекап — Експорт/Імпорт JSON у Налаштуваннях.

**Стек монорепо:** Express + better-sqlite3 + zod (бекенд); React + Vite + Tailwind +
@tanstack/react-query + @dnd-kit + recharts (фронт).

**API монорепо (`/api`):** `accounts`, `categories`, `transactions` (+ `/transfer`),
`analytics/by-category`, `analytics/summary`, `backup/export`, `backup/import`, `health`.

## 7. Робота з кодом у цьому репозиторії (для асистента/розробника)

- Працювати **тільки** в межах папки `TEST\MoneyMe`. Інші проєкти в `TEST` не чіпати.
- Машина зазвичай **без Node.js та інтернету** — `npm install`/`npm run dev` тут не запускаються;
  правити `MoneyMe.html` напряму й перевіряти у браузері.
- Фікси перевіряти у **реальному браузері** (а не лише в прев'ю) — прев'ю не ловить CSP/service worker.
- Репозиторій: **vh-ss/moneyme**, гілка `main`. Push працює лише з `http.sslBackend=schannel`
  (виставлено глобально). Після кожної зміни — `commit` + `push`.

## 8. Типові сценарії

- **Додати витрату:** Огляд → `−` (або Операції → нова витрата) → рахунок, сума, категорія, дата.
- **Переказ між рахунками:** перетягнути картку рахунку на інший (або Операції → Переказ) → задати
  комісію (% / фікс) і курс, якщо валюти різні.
- **Поставити бюджет:** Категорії → витратна стаття → Місячний/Річний бюджет (+ перенос невитраченого).
- **Налаштувати підписку:** Регулярні → сума, період, дата наступного списання.
- **Зробити бекап:** Налаштування → Експорт JSON (зберегти файл поза браузером).
- **Відновитися після помилки:** Налаштування → Точки відновлення → обрати знімок.
- **Увімкнути синхронізацію телефону:** виконати кроки розділу 5 / [SETUP.md](SETUP.md).
