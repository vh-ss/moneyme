# MoneyMe — розгортання та синхронізація через Google Drive

Щоб мати **авто-синхронізацію ПК ↔ телефон** із роботою офлайн, застосунок треба:
1. розмістити онлайн (безкоштовно — **GitHub Pages**, дає https);
2. підключити до **вашого Google Drive** через OAuth Client ID.

> Чому не можна «просто з файлу»? Google дозволяє вхід (OAuth) лише зі сторінок на `https://…`,
> а не з `file://` (подвійний клік). Тому потрібен хостинг. Сам файл `MoneyMe.html` і далі
> працює офлайн локально — просто без хмарної синхронізації.

---

## Крок 1. Розмістити на GitHub Pages

1. Створіть репозиторій на GitHub (наприклад `moneyme`). Для найзручнішого URL можна назвати
   репозиторій `ВАШ_ЛОГІН.github.io` — тоді сайт буде на `https://ВАШ_ЛОГІН.github.io/`.
2. Залийте туди **вміст папки MoneyMe** — обовʼязково файли:
   `index.html`, `sw.js`, `manifest.webmanifest`, теку `icons/` (а також `MoneyMe.html`).
   ```bash
   # з папки MoneyMe:
   git remote add origin https://github.com/ВАШ_ЛОГІН/moneyme.git
   git push -u origin main
   ```
3. У репозиторії: **Settings → Pages → Build and deployment → Source: Deploy from a branch**,
   гілка **main**, тека **/ (root)** → **Save**.
4. За ~1 хв сайт буде доступний за адресою:
   - `https://ВАШ_ЛОГІН.github.io/` (якщо репо названо `ВАШ_ЛОГІН.github.io`), або
   - `https://ВАШ_ЛОГІН.github.io/moneyme/` (якщо репо `moneyme`).

   Запишіть цю адресу — назвемо її **БАЗА**.

---

## Крок 2. Створити Google OAuth Client ID

1. Відкрийте **https://console.cloud.google.com/** → угорі **Create project** → назва «MoneyMe» → Create.
2. **APIs & Services → Library** → знайдіть **Google Drive API** → **Enable**.
3. **APIs & Services → OAuth consent screen**:
   - User type: **External** → Create.
   - Заповніть App name «MoneyMe», свій email у двох полях підтримки → Save and continue.
   - Scopes — нічого додавати не треба (застосунок запитає доступ сам) → Save and continue.
   - **Test users → Add users → додайте СВІЙ Google-акаунт** (той, з якого користуватиметесь) → Save.
   - Статус публікації лишіть **Testing** (для особистого використання верифікація Google не потрібна).
4. **APIs & Services → Credentials → Create Credentials → OAuth client ID**:
   - Application type: **Web application**, назва будь-яка.
   - **Authorized JavaScript origins → Add URI** і додайте ваш origin **без шляху**, напр.:
     - `https://ВАШ_ЛОГІН.github.io`
     - (за бажання для локальних тестів ще `http://localhost:4321`)
   - Redirect URI **не потрібен**. → **Create**.
5. Скопіюйте **Client ID** (виглядає як `1234567890-abc….apps.googleusercontent.com`).

---

## Крок 3. Увімкнути синхронізацію в застосунку

**На ПК:**
1. Відкрийте **БАЗУ** у браузері (Chrome/Edge).
2. **Налаштування → Синхронізація — Google Drive**:
   - вставте **Client ID** → **Зберегти ID**;
   - **Підключити Google Drive** → увійдіть тим самим Google-акаунтом (тест-юзером) → дозвольте доступ.
3. Статус має стати **✓ Синхронізовано**. Дані зберігаються у прихованій папці застосунку на вашому Drive
   (файл `moneyme-data.json`, невидимий у звичайному інтерфейсі Drive — лише для цього застосунку).

**На телефоні:**
1. Відкрийте ту саму **БАЗУ** у браузері.
2. Меню браузера → **Додати на головний екран** (Add to Home screen) — застосунок встановиться як іконка
   й працюватиме офлайн.
3. Відкрийте його → Налаштування → той самий **Client ID** → **Підключити Google Drive** → той самий акаунт.

Готово. Зміни на одному пристрої зʼявляються на іншому при відкритті/поверненні в застосунок
(і при появі мережі). Офлайн усе працює; синхронізація доганяє, коли зʼявляється інтернет.

---

## Як це працює / нюанси

- **Офлайн-first:** усі дані миттєво в `localStorage` пристрою. Drive — спільне хмарне сховище;
  при наявності мережі локальні зміни вивантажуються, а віддалені — підтягуються.
- **Вирішення конфліктів:** перемагає новіша версія (за міткою часу). Версію, що програла,
  застосунок зберігає у **Точках відновлення** — нічого не губиться.
- **Приватність:** використовується scope `drive.appdata` — застосунок бачить **лише свій** файл,
  не має доступу до решти вашого Drive.
- **Режим Testing:** для особистого акаунта верифікація не потрібна. Іноді (раз на кілька днів)
  Google може попросити повторний вхід — просто натисніть «Підключити» ще раз.
- **Оновлення застосунку:** після заливання нової версії на Pages зміни підхопляться при наступному
  відкритті (service worker оновлює кеш у фоні; інколи треба оновити сторінку двічі).

## Безпека даних

Client ID **не є секретом** — його можна тримати у відкритому коді/налаштуваннях (так і задумано для
веб-застосунків). Доступ захищає саме вхід у ваш Google-акаунт.
